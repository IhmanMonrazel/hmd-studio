# Impeccable Design Context — HMD Studio

## Design Context

### Users
Designers, creative developers, and studios evaluating HMD Studio as a potential creative partner. They arrive with a discerning eye — they will notice craft, pacing, and intention before they read a word of copy. Their job-to-be-done: quickly assess whether HMD operates at their level, then find a reason to reach out.

### Brand Personality
Bold, immersive, technically refined.

Voice: terse, authoritative, never over-explained. The work speaks; copy supports it.
Tone: cinematic — every interaction should feel like a frame cut, not a page load.
Emotional goal: leave visitors with a sense of being in the presence of something precise and powerful.

### Aesthetic Direction
- **Visual tone:** Premium editorial. Maximum contrast. Negative space used as a design element, not an absence.
- **Palette:** Strictly `#000000` / `#FFFFFF` / `#CC0000`. No grays in hero contexts. `rgba(0,0,0,0.38)` muted for supporting text only.
- **Typography:** Bebas Neue for display (condensed, commanding, all-caps). Inter Light (300) for body (legible, neutral, receding). No decorative fonts.
- **Motion:** GSAP scroll-driven reveals are the primary storytelling mechanism. Animations should feel inevitable, not performative — eases that decelerate into rest, never bouncy. Three.js objects rotate slowly and respond to mouse with lerp-smoothed tilt.
- **Density:** Sections breathe wide. `clamp()` padding. Never cramped. Whitespace is luxury.
- **References:** High-motion agency portfolios, cinematic title sequences, luxury brand sites.
- **Anti-references:** Startup landing pages, colorful gradient UI, card-heavy layouts, any interface that feels "SaaS".

### Design Principles

1. **Motion earns meaning.** Every animation must justify its existence by directing attention or communicating quality — no decorative motion for its own sake.

2. **Constraint is the aesthetic.** Three colors. Two fonts. One axis of contrast. Resist the urge to add. The power comes from what's withheld.

3. **Craft at the pixel level.** Letter-spacing, line-height ratios, and timing curves are not defaults — they are deliberate. `0.88` line-height on display type. `0.22em` tracking on uppercase labels. `0.25s` transitions minimum.

4. **Sections are scenes.** Each scroll section is a discrete cinematic moment with its own pacing and reveal choreography. Treat transitions between sections as cuts, not fades.

5. **Technical refinement is visible.** The stack (Three.js, GSAP, Remotion, Rails) is not hidden — it's the signal. Code quality and render quality are the same thing here.

## Stack Reference
- **Framework:** Ruby on Rails 7.1 + React (via esbuild) + Stimulus + Turbo
- **Animation:** GSAP (ScrollTrigger) + Three.js (tribal canvas motif)
- **Video:** Remotion (carousel export)
- **Fonts:** Bebas Neue (display), Inter (body) — loaded via Google Fonts or CDN
- **CSS convention:** BEM-ish, single `application.css` entry, CSS custom properties in `:root`

```css
--black:        #000000

--font-body:    'Inter', sans-serif
--nav-h:        72px
```

## Animation Conventions
- Use `gsap.set()` + `onEnter` + `gsap.to()` — never `gsap.from()` + ScrollTrigger (visibility bug)
- `ScrollTrigger.batch()` for staggered group reveals
- `scrub: true` on timelines for scroll-linked 3D
- Never set `opacity: 0` in CSS on GSAP-animated elements — GSAP owns initial state
- Three.js canvas appended via JS to `this.element`, `position: absolute; bottom: 40px; right: 40px`
